## 11. 评估体系（RAG Evaluation）

> **背景：** RAG 项目最难的部分不是搭建，而是评估与持续优化。没有评估体系，优化方向将无从判断。

### 11.1 评估方法

  - [ ] 需要 Human-in-the-loop 评估（专家逐条标注）
  - [ ] 需要抽样评估（随机抽取 X% 进行人工复核）
  - [ ] 无需人工评估

- **自动化评估框架：**
  - [ ] **Ragas**（基于 LLM-as-Judge，主流选择）
  - [ ] **TruLens**（支持多种评估维度，有 UI 界面）
  - [ ] **G-Eval**（基于 GPT-4 评分）
  - [ ] **ARES**（适合大规模自动评估）

### 11.2 测试集建设

- **是否需要构建专项 QA 测试集：**
  - [ ] 是，需要构建（包含：问题、标准答案、参考上下文三元组）

  - [ ] 使用 LLM 自动生成测试集（如 Ragas 的 TestsetGenerator）

| 测试集维度 | 要求 |

### 11.3 负向测试与幻觉容忍度

> **幻觉（Hallucination）定义：** 模型回答的内容在检索到的知识库中找不到依据，是凭空"编造"的。

- **当知识库中没有相关答案时，系统应：**
  - [ ] **明确拒答**："根据现有知识库，我无法回答此问题"（推荐，避免幻觉）
  - [ ] **尝试回答并标注不确定性**："以下回答可能不准确，仅供参考..."
  - [ ] **混合策略**（可配置，由管理员决定）

- **幻觉容忍度指标：**

| 指标 | 目标值 |
|------|--------|
| 无依据回答率（Hallucination Rate） | ≤ __% |
| 知识库外问题拒答率 | ≥ __% |
